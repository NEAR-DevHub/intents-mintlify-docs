Distribution Channels are the user-facing applications, wallets, and exchanges that make NEAR Intents accessible to end users. They handle intent creation, management, and asset delivery, while [Market Makers](/integration/market-makers/introduction) handle fulfillment and pricing.

## What is a Distribution Channel?

A distribution channel is any application that helps users create and execute intents:

- **Wallets** - Let users swap tokens directly from their wallet interface
- **DEX aggregators** - Route trades through NEAR Intents for better pricing
- **DApps** - Integrate cross-chain swaps into your application
- **Exchanges** - Offer intent-based trading to your users

## 1Click Swap API

The fastest way to integrate NEAR Intents. The 1Click API handles all the complexity:

- **Simple REST endpoints** for intent creation and management
- **Automatic solver discovery** and competitive pricing
- **Built-in transaction handling** and status tracking
- **Cross-chain support** for seamless multi-chain swaps

### How it works

```
┌─────────────────┐     1. Request quote      ┌─────────────────┐
│   Your App      │ ────────────────────────▶ │   1Click API    │
│                 │                           │                 │
│                 │ ◀──────────────────────── │                 │
└─────────────────┘     Quote + deposit addr  └─────────────────┘
        │                                              │
        │ 2. User deposits                             │
        │    to address                                │
        ▼                                              ▼
┌─────────────────┐                           ┌─────────────────┐
│  Source Chain   │                           │  Market Makers  │
│  (ETH, BTC...)  │                           │  execute swap   │
└─────────────────┘                           └─────────────────┘
                                                       │
                                                       │ 3. Tokens delivered
                                                       ▼
                                              ┌─────────────────┐
                                              │  Destination    │
                                              │  (user wallet)  │
                                              └─────────────────┘
```

### Quick start

<Steps>
  <Step title="Get an API key">
    Register on the [Partner Dashboard](https://partners.near-intents.org/) to get your JWT token. Authenticated requests avoid the 0.1% platform fee.
  </Step>

  <Step title="Request a quote">
    Call the [`/v0/quote`](/api-reference/endpoint/quote) endpoint with the user's swap details. You'll receive pricing and a unique deposit address.
  </Step>

  <Step title="User deposits tokens">
    The user sends tokens to the deposit address. 1Click automatically begins the swap.
  </Step>

  <Step title="Monitor status">
    Optionally poll [`/v0/status`](/api-reference/endpoint/status) to track progress and display updates to your user.
  </Step>
</Steps>

**Result:** Tokens are delivered to the user's destination address, or automatically refunded if the swap fails.

## Earning fees

Distribution channels can earn fees on every swap. Configure fee collection when requesting quotes:

```json
{
  "appFees": {
    "recipient": "your-wallet.near",
    "fee_bps": 50
  }
}
```

This charges a 0.5% fee (50 basis points) from the input token, sent to your wallet.

For high-volume integrations, use [fee aggregation](/api-reference/fee-config#fee-aggregation) to automatically convert all fees into a single token.
